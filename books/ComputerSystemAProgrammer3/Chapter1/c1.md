## 第一章 计算机系统的漫游

本章节对计算机语言的执行过程进行了简要简析

---

### 1.1 信息就是位+上下文

计算机中的文本字符通常用 ASCII 码的整数值表示，每个整数值是一个**唯一的单字节**，以下给出示范：

```c
//hello.c
#include <stdio.h>
int main{
    printf("Hello world\n")
    return 0;
}
```

该程序文件存储在文件名为 `hello.c` 的文件中，以字节序列的方式存储文本文件，如 `#` 对应的 ASCII 为 35，每行由一个 `\n` 结尾，其 ASCII 为 10.

上诉全部由 ASCII 存储的文件称为 **文本文件**，其他类型的文件成为**二进制文件**

---

### 1.2 程序编译过程

`hello.c` 的源程序经过预处理器 (CPP)，转化为汇编语言，再经编译器和汇编器转化为可执行的目标程序（可执行目标文件）`hello`，最终存储在内存中，这个转化过程可以由_编译器驱动程序_直接转化

```shell
linux>gcc -o hello hello.c
```

因此，在 Unix 中，我们可以运行以下命令直接运行 `hello.c`

```shell
linux> ./hello
hello.world
```

接下来我们将根据与硬件的结合来查明 `hello.c` 的编译过程

---

### 1.3 程序硬件描述

系统硬件可以分为：总线、I/O设备、主存和处理器

1. 总线：相当于一条管道，负责运送系统的字节流
2. I/O设置：通过**控制器**或**适配器**与I/O总线相连接，如USB控制器(鼠标、键盘)、图形适配器(显示器)等专门与I/O设备交互
   1. 控制器：I/O设备或系统主板本身的芯片组
   2. 适配器：主板上的插槽卡
